# Project Evolution Timeline

Track how this project evolved over time.

## How to Use

This file is created by the `/forensic` command, which analyzes git history to:
- Construct timeline from commits
- Identify architectural inflection points
- Track pattern emergence
- Find file hotspots (complexity centers)
- Map author knowledge distribution

Run `/forensic` to generate this timeline automatically.

## Manual Timeline

If this is a new project or you want to document evolution manually:

```markdown
## Phase: [Phase Name] (YYYY-MM to YYYY-MM)

**Major Features**:
- Feature 1
- Feature 2

**Key Changes**:
- Change 1
- Change 2

**Patterns Introduced**:
- Pattern name (see patterns.md)

**Decisions Made**:
- Decision (see decisions.md)
```
